# 智弈量化比赛 PPT 策划（14页版）

## 一、整体建议

这套 PPT 建议按照 `14 页` 来做，适合比赛展示与答辩使用，既能把项目介绍完整，也能给“亮点、创新点、演示视频”留出足够空间。  

整套 PPT 的表达主线建议固定为：

`为什么做 -> 做了什么 -> 怎么实现 -> 实际效果 -> 有什么创新 -> 为什么有价值`

### 整体视觉风格

建议采用“金融科技 + 桌面智能工具”的视觉风格。

- 主背景色：深蓝 `#0B1F3A`
- 辅助色：科技青 `#16C2D5`
- 强调色：亮金 `#F4B942`
- 正向色：绿色 `#22C55E`
- 风险色：红色 `#EF4444`

### 统一设计语言

- 页头统一用大标题 + 短副标题
- 每页只保留一个主视觉中心，不要平均铺满
- 内容尽量使用卡片、流程箭头、模块框图、数据标签
- 截图优先来自真实系统界面，减少纯概念化插图
- 所有页面尽量统一圆角、发光边框、细网格背景和科技线条元素

### 推荐优先截图页面

- `Dashboard`：适合封面、项目总览
- `AI Analysis`：适合展示 AI 分析能力
- `Indicator Analysis`：适合展示指标编辑和回测
- `Trading Assistant`：适合展示策略创建、运行与交易记录
- `Portfolio`：适合展示资产监测、预警与组合管理

## 二、推荐的 14 页结构

## 第 1 页：封面页

### 页面目标

让评委第一眼知道项目名称、项目定位和技术气质。

### 标题文案

智弈量化  
本地优先的桌面量化分析与交易辅助平台

### 副标题文案

打通“市场分析 - 策略研究 - 回测验证 - 运行监控”的完整闭环

### 页面文案要点

- 面向个人用户与量化初学者，`V1` 版本聚焦单人本地使用
- 集成行情、AI、回测、策略、监控五类核心能力
- 兼具产品完整度与工程实现能力

### UI 设计建议

封面采用深蓝渐变背景，中间放电脑或桌面应用 mockup，屏幕内展示系统主界面。标题放左上或中上区域，右下角可以加一句简短口号。

### 图案与组件建议

- 笔记本电脑或应用窗口 mockup
- K 线图、数据卡片、AI 节点线条
- 品牌 Logo：`brand-full.png` 或 `brand-mark.png`

## 第 2 页：项目背景与问题提出

### 页面目标

说明项目为什么要做，让评委先认同问题，再接受方案。

### 标题文案

为什么我们需要智弈量化？

### 页面文案要点

- 量化研究通常需要在多个平台间反复切换，流程割裂
- 传统量化工具专业门槛高，不适合普通学生和个人用户
- 数据、策略、交易和监控分散，难以形成统一工作流
- 用户需要一个更低门槛、更完整、更适合个人使用的量化工作台

### UI 设计建议

左侧用 3 张痛点卡片，右侧用一条“碎片化工具链”流程图，对比传统方式的复杂度。

### 图案与组件建议

- 痛点卡片：功能分散、门槛高、管理难
- 多工具串联流程图
- 断裂箭头、警示图标、云端图标

## 第 3 页：项目定位与核心目标

### 页面目标

明确项目到底是什么，面向谁，解决什么问题。

### 标题文案

项目定位：面向个人用户的本地量化工作台

### 页面文案要点

- 智弈量化是一款本地部署优先的桌面量化分析与交易辅助平台
- 面向量化初学者、金融科技爱好者、个人研究型用户
- 目标是降低量化研究门槛，整合分析、验证和跟踪流程
- `V1` 产品形态聚焦单人本地使用，强调学习研究与工程实践价值，不以投资建议为导向

### UI 设计建议

中间放“项目定位”主圆形模块，两侧放目标用户画像卡片和目标价值卡片。

### 图案与组件建议

- 用户画像卡片
- 靶心图或定位图
- 标签组件：本地化、一体化、智能化、低门槛

## 第 4 页：项目整体功能概览

### 页面目标

用一页把项目做的事情讲全，建立整体认识。

### 标题文案

智弈量化做了什么？

### 页面文案要点

- 标的搜索、自选管理、图表分析和技术指标展示
- AI 分析与历史记录管理
- 自定义指标、参数化指标与策略回测
- 策略创建、运行与交易辅助
- 组合持仓、监测、预警与本地单用户设置管理

### UI 设计建议

建议做“中心平台 + 周围五大模块”的总览图，也可以采用 2 行 3 列模块卡片布局。

### 图案与组件建议

- 模块卡片
- 功能图标：行情、AI、回测、策略、资产
- 简化版系统缩略截图

## 第 5 页：项目业务闭环

### 页面目标

体现项目不只是功能堆叠，而是完整流程闭环。

### 标题文案

从研究到监控，形成完整业务闭环

### 页面文案要点

- 搜索市场与标的，查看图表与行情
- 使用技术指标和 AI 分析形成判断
- 编写策略并进行回测验证
- 运行策略并查看执行状态与交易记录
- 持续监控资产并接收预警提醒

### UI 设计建议

建议使用横向 5 步流程图或闭环环形图，中间突出“智弈量化平台”。

### 图案与组件建议

- 五步流程节点
- 箭头闭环
- 图表、AI、代码、策略、铃铛图标

## 第 6 页：技术架构与实现思路

### 页面目标

体现计算机设计类比赛需要的技术深度与工程思维。

### 标题文案

技术架构：桌面宿主 + 前端界面 + 本地量化引擎

### 页面文案要点

- `Tauri` 负责桌面应用封装，并以 sidecar 方式启动本地 Python 引擎
- `Vue` 负责桌面交互界面和功能页面呈现
- `Python` 负责行情处理、AI 分析、指标计算、回测和策略执行
- `SQLite` 负责本地数据持久化与历史记录管理
- 核心流程、配置和历史记录在本地编排与存储，AI/新闻/宏观等能力通过可配置接口接入
- 架构兼顾桌面体验、数据分析能力和本地部署优势

### UI 设计建议

建议做分层架构图，上层是用户界面，中层是 `Vue + Tauri`，下层是本地 Python 引擎与 `SQLite`，外围接行情源、AI 模型和交易接口。

### 图案与组件建议

- 分层框图
- 双向箭头
- 技术标签 Chip：`Tauri 2`、`Vue 2`、`Python`、`SQLite`
- 外部模块标签：LLM、市场数据、IBKR、MT5、交易所

## 第 7 页：核心模块展示一，市场分析与 AI 分析

### 页面目标

展示项目最容易被理解、最具产品感的能力。

### 标题文案

核心模块一：市场分析 + AI 辅助解读

### 页面文案要点

- 支持查看市场行情、图表和技术指标
- AI 分析会综合技术、宏观、新闻和情绪信息生成结果
- 输出不仅有判断，还包括理由、风险、置信度和建议
- 分析历史可保留，便于回顾与复盘

### 展示提示

答辩时建议把“本地优先”解释为“本地部署、本地编排、本地存储”，不要表述成“完全离线 AI”。

### UI 设计建议

推荐放一张 AI 分析页面大截图，旁边用 4 个说明卡解释“输入了什么、输出了什么”。

### 图案与组件建议

- 大尺寸界面截图
- AI 结果卡片：决策、风险、置信度、建议
- 数据输入标签：行情、新闻、宏观、指标

## 第 8 页：核心模块展示二，指标分析、回测与策略助手

### 页面目标

强调项目不是只会“看”，还支持“研究与验证”。

### 标题文案

核心模块二：指标分析、回测验证与策略运行

### 页面文案要点

- 支持自定义指标和参数化配置
- 支持策略回测并展示收益率、回撤、胜率等结果
- 支持策略创建、管理和运行，并展示执行状态与交易记录
- 让用户先验证策略，再进入后续运行与跟踪流程

### UI 设计建议

这一页可以做左右双栏。左边放指标分析或回测截图，右边放交易助手截图，中间用箭头连接“研究 -> 验证 -> 运行”。

### 图案与组件建议

- K 线图与指标图层
- 回测结果卡：收益率、最大回撤、胜率、夏普比率
- 策略列表卡、运行状态卡

## 第 9 页：核心模块展示三，资产监测与预警机制

### 页面目标

展示项目闭环的最后一环，说明系统具备持续服务用户的能力。

### 标题文案

核心模块三：资产监测、预警提醒与持续跟踪

### 页面文案要点

- 支持持仓管理与组合跟踪
- 支持价格预警、盈亏提醒和 AI 监控任务
- 支持基于持仓的 AI 分析和通知能力
- 把策略结果和资产状态连接起来，形成持续监测链路

### UI 设计建议

推荐使用“三卡布局”：持仓卡、预警卡、监控报告卡。中间放一张 `Portfolio` 页面截图。

### 图案与组件建议

- 组合持仓面板
- 预警铃铛图标
- 监控任务卡、通知卡、收益变化小折线图

## 第 10 页：系统演示视频页

### 页面目标

把前面讲的内容转化成直观动态展示，这是整套 PPT 的中段高潮页。

### 标题文案

项目演示：智弈量化完整使用流程

### 页面文案要点

- 建议视频时长控制在 `60-90 秒`
- 视频内容应覆盖“分析 -> 回测 -> 策略 -> 监控”主链路
- 用真实操作录屏体现产品完成度和工程实现度

### 推荐视频脚本

1. 打开首页仪表盘，展示系统整体界面  
2. 进入 AI 分析页，选择标的并展示分析结果  
3. 进入指标分析页，展示指标编辑与回测结果  
4. 进入交易助手页，展示策略创建或策略列表  
5. 进入资产监测页，展示持仓、预警或监控信息  
6. 最后回到总结页或系统总览，形成闭环印象

### UI 设计建议

这一页中间放 16:9 视频播放器区域，四周不要堆太多文字。左下或右下可以放“演示步骤标签条”，例如“AI 分析 / 回测 / 策略 / 监控”。

### 图案与组件建议

- 视频播放器 mockup
- 播放按钮、时间轴、章节标签
- 步骤标签条
- 发光边框和操作焦点框

## 第 11 页：创新点一，桌面 Sidecar 架构

### 页面目标

突出项目区别于普通网页工具和碎片化工具的根本优势。

### 标题文案

创新点一：Tauri + Python Sidecar 的本地桌面量化架构

### 页面文案要点

- 采用桌面应用形态，前端界面与本地量化引擎集成在同一系统内
- `Tauri` 负责桌面宿主，启动本地 Python 引擎处理分析、回测与策略逻辑
- 配置、记录和历史结果落地本地数据库，适合个人研究与持续调试
- 体现了桌面端工程集成能力，而不只是普通网页展示

### UI 设计建议

建议做“前端界面 - Tauri 宿主 - Python 引擎 - SQLite”分层结构图，也可以配一张真实桌面界面截图。

### 图案与组件建议

- 分层框图
- 本地引擎图标、本地数据库图标、接口连接箭头
- 集成工作台面板

## 第 12 页：创新点二，多维 AI 分析与分析记忆

### 页面目标

把 AI 能力讲具体，说明它不是简单接入模型，而是结合业务做了结构化设计。

### 标题文案

创新点二：多维 AI 分析，不只给结论，还保留分析记忆

### 页面文案要点

- AI 分析会综合行情、技术指标、宏观环境和新闻信息
- 输出结构化结果，包括结论、风险、理由、评分和交易建议
- 分析结果会保存为历史记录，便于后续复盘和连续分析
- 相比单次问答，更接近面向业务流程的智能分析模块

### UI 设计建议

中间放 AI 分析引擎图，周围放输入源，右侧放输出结果卡和历史记录时间轴。

### 图案与组件建议

- AI 中心节点
- 输入源卡片：价格、指标、宏观、新闻、历史
- 输出卡片：决策、评分、风险、理由
- 时间轴或记忆库图形

## 第 13 页：创新点三，高精度回测与扩展能力

### 页面目标

突出项目不是单点功能，而是兼顾闭环能力和扩展空间。

### 标题文案

创新点三：高精度回测能力与多市场扩展基础

### 页面文案要点

- 回测模块可展示收益率、最大回撤、胜率、夏普比率等关键指标
- 对部分场景可采用更高精度的执行时间框架，提高回测可信度
- 后端已具备股票、外汇、加密等多市场扩展基础
- 已接入或预留 `IBKR`、`MT5` 与交易所接口，便于后续扩展更多交易场景

### UI 设计建议

建议采用“回测指标卡 + 多市场扩展图”的组合布局。左边强调回测能力，右边展示接口扩展空间。

### 图案与组件建议

- 回测指标卡
- 扩展分支图
- 市场标签：股票、外汇、Crypto
- 接口标签：IBKR、MT5、Exchange

## 第 14 页：项目价值总结与未来展望

### 页面目标

收束前面所有内容，让评委记住这个项目为什么有意义。

### 标题文案

项目价值与总结

### 页面文案要点

- 智弈量化解决了量化工具分散、复杂、门槛高的问题
- 项目形成了“本地部署 + 一体化 + 智能辅助”的产品方案
- 项目不仅有功能完整度，也体现了较强工程实现能力
- 项目具备继续扩展多市场、多策略和更强 AI 能力的空间

### 收尾文案

我们希望智弈量化不仅是一个比赛作品，更是一个具有真实应用潜力的个人量化桌面平台。

### UI 设计建议

左边用三大关键词总结项目价值，右边展示未来拓展方向，底部可以再次放项目 Logo 和一句总结性口号。

### 图案与组件建议

- 三大关键词卡片：本地部署、一体化、智能辅助
- 路线延展图
- Logo、发光边框、简洁数据背景
