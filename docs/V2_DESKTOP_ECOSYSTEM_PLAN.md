# 智弈量化 V2 生态规划

V2 面向智弈量化的下一阶段生态能力，在 V1 单人桌面产品稳定之后，围绕指标、策略、研究模板与资产包建立可分享、可导入、可验证的扩展体系。

## 目标

- 让指标、策略、研究模板成为标准化资产包
- 支持本地导入、导出、版本化与签名校验
- 为后续目录服务、同步能力和扩展服务预留清晰边界

## 设计原则

- V1 运行时继续保持纯桌面、本地优先
- 生态能力与桌面运行时分层设计，边界清晰
- 先定义资产格式与可信机制，再扩展联网能力
- 每一项在线能力都以独立服务方式设计，不侵入 V1 主运行时

## 建议阶段

1. 定义资产模型：指标包、策略包、研究模板、元数据、版本信息。
2. 定义导入导出标准：文件结构、签名、兼容版本、清单说明。
3. 先实现离线分享、本地库管理与资产校验。
4. 在桌面端建立统一的资产浏览、安装、升级与回滚体验。
5. 如需联网目录、同步或分发能力，再独立建设扩展服务。
