# 智弈量化项目说明

## 1. 项目定位

智弈量化是一款面向个人用户的本地桌面量化分析与交易辅助平台。`V1` 版本主要面向单人本地使用。项目希望把原本分散在多个软件和网页中的能力整合到同一个系统中，让用户能够在一个界面内完成标的查看、技术分析、AI 辅助解读、指标编写、策略回测、策略运行和资产监控，形成较完整的量化研究闭环。

## 2. 项目为什么要做

当前常见的量化工具存在几个明显问题。  
第一，功能割裂，用户往往需要在行情软件、图表工具、策略平台和交易终端之间来回切换。  
第二，专业门槛较高，很多系统更适合专业开发者，普通学生和个人投资者很难快速上手。  
第三，很多平台依赖云端服务，用户的数据、策略和账户配置不容易统一管理。  

因此，我们设计了一个本地优先的一体化桌面工作台，希望降低量化分析和策略研究的使用门槛，让用户更方便地完成“分析市场、验证策略、跟踪结果”的全过程。

## 3. 项目主要做什么

本项目围绕量化研究和交易辅助场景，提供以下核心能力：

- 标的搜索、自选管理、K 线图表展示和技术指标分析
- AI 分析与分析历史记录，帮助用户更快理解市场信息
- 自定义指标、参数化指标、策略回测和回测结果查看
- 策略创建、策略运行和交易辅助执行
- 组合持仓、预警提醒和持续监控
- 本地单用户账号、设置、凭据和数据管理

从功能链路上看，它不是单一的行情查看工具，也不是单一的交易终端，而是一个把研究、验证、执行和管理连接起来的量化工作平台。

## 4. 用户如何使用这个系统

用户可以先搜索感兴趣的市场或标的，查看行情和图表，结合技术指标与 AI 分析快速形成判断；随后可以编写或调整指标和策略，对策略进行回测，观察历史表现；在验证有效后，用户还可以继续在系统中运行策略、跟踪持仓、查看交易记录，并通过预警和监控模块持续观察资产变化。  

也就是说，这个项目打通了从“看市场”到“做策略”，再到“跟踪运行结果”的主要流程，适合个人研究与交易辅助场景。

## 5. 项目技术实现思路

项目采用“桌面宿主 + 前端界面 + 本地分析引擎”的分层结构：

- `Tauri` 负责桌面应用封装，并以 sidecar 方式启动本地 Python 引擎
- `Vue` 负责前端交互界面与功能页面展示
- `Python` 负责行情处理、指标计算、回测、AI 分析和策略执行
- `SQLite` 负责本地数据存储与管理

其中，桌面程序会在本地启动 Python 引擎，前端界面向本地接口发起请求，再把分析、回测和监控结果直观展示给用户。主要流程、配置和历史记录在本地编排与存储；部分 AI、新闻和宏观数据能力则通过可配置的外部接口获取。这种架构兼顾了桌面应用的交互体验和 Python 在数据分析、量化计算方面的优势。

## 6. 项目特色与创新点

- 一体化设计：将市场分析、策略回测、策略运行和资产管理整合在一个桌面平台中
- 本地编排：桌面端直接启动本地 Python 引擎，并将配置、记录和历史结果存储到本地数据库
- 智能辅助：AI 模块输出结构化分析结果，并保留分析历史，便于回顾与复盘
- 可扩展性强：后端已具备多市场、多接口接入基础，可继续扩展更多交易场景
- 面向实际应用：不仅能展示数据，还能服务于研究、验证、运行和监控流程

## 7. 适合在 PPT 中强调的价值

如果用于比赛展示，可以重点突出以下三点：

- 它解决的是量化工具“分散、复杂、门槛高”的现实问题
- 它强调“本地部署 + 一体化 + 智能辅助”的产品思路
- 它覆盖从研究到运行再到监控的主要业务链路，具有实际应用价值

## 8. 一句话概括

智弈量化是一个面向个人用户、以本地部署为主的桌面量化工作台，用一个系统串联“市场分析 - 策略研究 - 回测验证 - 运行监控”的主要流程。
